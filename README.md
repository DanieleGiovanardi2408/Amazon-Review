## What this is

A semantic search system for Amazon Health & Personal Care products.  
Instead of matching exact keywords, it finds products by **meaning** — using dense vector embeddings built from product metadata and user reviews.

**Example:**
```
Query: "affordable moisturizer for sensitive skin"
→ Finds: Colonial Dames Vitamin E Cream $16.49 ⭐4.6
   (not a $119 product just because it matches keywords)
```

## Setup

### 1. Clone and install dependencies

```bash
git clone <repo-url>
cd discoverai
pip install -r requirements.txt
python -m spacy download en_core_web_sm
```

### 2. Download the dataset

From [McAuley-Lab/Amazon-Reviews-2023](https://huggingface.co/datasets/McAuley-Lab/Amazon-Reviews-2023), download the **Health_and_Personal_Care** category:

- `meta_Health_and_Personal_Care.json` — product metadata
- `Health_and_Personal_Care.json` — reviews

Place both files in `data/`.

### 3. Update DATA_DIR

In each notebook, update `DATA_DIR` to point to your `data/` folder:

```python
DATA_DIR = "/path/to/discoverai/data"
```

## Key design decisions

### Data cleaning (notebook 01)

| Decision | Why |
|---|---|
| Min 3 verified reviews per product | Fewer reviews = noise, not signal |
| Balanced rating selection: 5×5⭐ + 3×4⭐ + 2×3⭐ + 3×2⭐ + 3×1⭐ | Raw data is 61% 5-star — this brings it to 48% |
| Min 25 tokens per review | 48.7% of raw reviews are < 20 tokens ("Love it!!") |
| FORM field at top of product text | Disambiguates shampoo vs capsule vs cream |
| 18 semantic fields from `details` | 98% product coverage vs 29% for `description` |
| Cap 400 tokens per product | MPNet hard limit is 512 subword tokens |

### Embedding (notebook 03)

| Decision | Why |
|---|---|
| `all-mpnet-base-v2` | STS benchmark 72.4 vs 68.1 for MiniLM — better quality |
| Two-tower architecture | Separates product metadata and review signals; respects 512-token limit |
| Weighted review average | `weight = 0.7 × log1p(helpful_vote) + 0.3 × \|rating−3\|/2` |
| Dynamic alpha per product | Weak review signal → product embedding dominates |
| Bias correction | Rare reviews (e.g. negative in mostly-positive products) get boosted |
| `IndexFlatIP` (FAISS) | Exact search; < 5ms for 11k products; no approximation needed |

### Search (notebook 04)

| Decision | Why |
|---|---|
| Query parser | Strips price words before encoding — fixes the $119 affordable moisturizer bug |
| Min rating 3.5 default | Removes low-quality results (fixed the 3.2-star melatonin bug) |
| Dosage filter | "vitamin C 1000mg" finds 1000mg products, not 500mg |
| Synonym expansion | "help me sleep" → "sleep aid melatonin supplement" |
| Negation filter | "without melatonin" → excludes products with "melatonin" in title |
| Adaptive β_quality | 0.20 when similarity is weak (< 0.65), 0.12 otherwise |

## Known limitations

- **Price data missing for 77.5% of products** — price filters only work on 22.5% of the catalogue
- **No ground truth for retrieval evaluation** — quality measured qualitatively on 20 test queries (13/20 correct)
- **Negation filter is rule-based** — "without melatonin" checks titles only, not ingredient lists
- **BART summaries vary in quality** — products with generic reviews get less informative summaries
